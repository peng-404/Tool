# Win-Deep-Cleaner

这是一个用于清理 Windows 空间占用的桌面脚本工具。它主要做三件事：

- 扫描常见缓存、开发缓存、非活跃安装包和大文件，生成审查清单
- 在你确认之后执行删除，或禁用不必要的开机自启动项
- 分析 `C:` 所在磁盘的分区布局，判断是否具备扩容条件

这个项目默认采取保守策略：先扫描、再审查、再执行；高风险路径会被拦截，分区功能只做分析，不自动改盘。

## 当前能力

### 阶段一：扫描并生成审查清单

程序会完成以下事情：

- 调用 `dism` 和 `cleanmgr` 做系统原生清理
- 扫描第三方启动项
- 扫描多浏览器、多应用缓存与更多开发环境缓存
- 扫描下载、桌面、文档目录里半年未活跃的安装包、压缩包、大文件和部分深层子目录文件
- 为每条候选路径标注来源、类型、最近活动时间、大小
- 自动硬拦截高风险路径，以及疑似系统/硬件/网络必需的启动项
- 在桌面生成 `AI-Review-List.txt` 和 `AI-Review-Metadata.json`

这个阶段不会执行删除。

### 阶段二：勾选并执行

你可以把 `AI-Review-List.txt` 发给 AI，也可以自己手动删掉不想处理的行。保存后回到程序主界面，点击“开始阶段二”：

- 程序会重新读取你保留下来的路径
- 把它们做成勾选式列表，而不是直接处理
- 你可以再次取消勾选
- 文件类候选会执行删除；启动项候选会执行“禁用开机自启”
- 可以选择“先备份”，并让文件删除优先进入回收站
- 执行完成后会自动生成 TXT + JSON 报告

### 分区分析：只分析，不改盘

主界面提供 `Analyze C: Layout` 按钮，用来检测：

- `C:` 当前是否可以直接扩容
- `C:` 右侧是哪一个分区在阻挡扩容
- 是否存在恢复分区 / OEM 分区等常见阻挡项
- 只能从“同一块物理磁盘”上挪空间给 `C:`，不能直接从另一块磁盘给系统盘加空间

这个功能只会生成分析报告，不会自动修改分区。

## 使用方法

1. 双击运行 [Run-Win-Deep-Cleaner.bat](./Run-Win-Deep-Cleaner.bat)。
   也可以继续使用 [双击运行我.bat](./双击运行我.bat)。
2. 点击 `Start Phase One` 生成审查清单。
3. 查看桌面上的 `AI-Review-List.txt` 和 `AI-Review-Metadata.json`。
4. 删除你不想处理的条目，只保留要继续执行的行。
5. 保存 TXT 后回到主界面，点击 `Start Phase Two`。
6. 在勾选界面里再次确认，然后执行删除或启动项禁用。
7. 如果只想判断 `C:` 是否具备扩容条件，可以点击 `Analyze C: Layout`。

## 当前扫描范围

- 第三方启动项
- Edge / Chrome / Brave / Firefox 多 profile 缓存
- 微信缓存、QQ 临时目录、Slack / Discord / Teams / VS Code 缓存
- CrashDumps、WER 报告、Windows 缩略图缓存、系统临时目录
- HuggingFace、PyTorch、pip、Conda、npm、Yarn、pnpm、NuGet、Gradle、Maven、Cargo、Go、Playwright、JetBrains 等开发缓存
- 下载目录半年未活跃的安装包、压缩包、较大文件，以及最多两层子目录中的同类文件
- 桌面半年未活跃的安装包、压缩包、较大文件，以及最多一层子目录中的同类文件
- 文档目录半年未活跃的安装包、压缩包，以及最多一层子目录中的同类文件
- 当前用户和所有用户的常见启动注册表项、启动文件夹项

## 当前安全机制

- Windows 目录硬拦截
- 驱动相关目录硬拦截
- 系统盘根目录硬拦截
- 疑似系统 / 硬件 / 网络必需的启动项默认拦截
- 执行前再次检查路径是否仍然安全
- 可选备份
- 可选回收站模式
- 自动生成执行报告
- 分区功能只做检测与报告，不自动改盘

## 项目结构

- `Win-Deep-Cleaner.ps1`: 入口脚本，只负责提权、加载模块、启动界面
- `Modules/AppContext.ps1`: 应用上下文和共享路径 / 颜色配置
- `Modules/Core.ps1`: 通用 UI、路径、格式化、进程等待工具
- `Modules/Candidates.ps1`: 候选项模型、扫描逻辑、审查清单与元数据处理
- `Modules/Actions.ps1`: 删除、启动项禁用、备份、执行报告
- `Modules/Partition.ps1`: `C:` 分区扩容分析与报告
- `Modules/Workflow.ps1`: phase one / phase two 工作流编排
- `Modules/UI.ps1`: 主界面和执行确认窗口

## 不建议直接处理的内容

- 微软系统核心目录
- 驱动目录
- 你不了解用途的软件安装目录
- 普通文档、照片、工作资料这类个人文件
- 需要复杂依赖分析的大型开发环境
