# Win-Deep-Cleaner

这是一个面向普通 Windows 用户的两阶段清理候选审查器。第一阶段负责扫描并生成带来源标签的审查清单；第二阶段只会在你审核过后，以勾选式界面执行删除，并自动保留备份与报告。

项目的核心思路不是“脚本自动乱删”，而是把扫描、判断、执行、审计拆开处理。

## 项目定位

- 先做“清理候选发现器”，再做“有审计能力的执行器”
- 优先处理高频问题：第三方自启、常见缓存、开发缓存、半年未活跃的安装包与大文件
- 让不懂电脑的人也能先看清楚候选来源，再决定删不删
- 危险操作尽量后置，并在执行前再加一层硬拦截、备份和报告

## 当前能力

### 阶段一：扫描并生成审查清单

程序会完成以下事情：

- 调用 `dism` 和 `cleanmgr` 做系统原生清理
- 扫描第三方启动项
- 扫描常见应用缓存与开发环境缓存
- 扫描下载、桌面、文档目录里半年未活跃的安装包、压缩包和大文件
- 为每条候选路径标注来源、类型、最近活动时间、大小
- 自动硬拦截高风险路径，例如 Windows 目录、驱动目录、系统盘根目录
- 在桌面生成 `AI-Review-List.txt` 和 `AI-Review-Metadata.json`

这个阶段不会执行删除。

### 阶段二：勾选并执行

你可以把 `AI-Review-List.txt` 发给 AI，也可以自己手动删掉不想处理的行。保存后回到程序主界面，点击“开始阶段二”：

- 程序会重新读取你保留下来的路径
- 把它们做成勾选式列表，而不是直接删除
- 你可以再次取消勾选
- 可以选择“先备份”与“优先移动到回收站”
- 执行完成后会自动生成 TXT + JSON 报告

## 使用方法

1. 双击运行 [双击运行我.bat](./双击运行我.bat)。
2. 在主界面点击 `Start Phase One`。
3. 等待桌面生成 `AI-Review-List.txt` 和 `AI-Review-Metadata.json`。
4. 把 TXT 内容交给 AI 或自己筛选，只保留你想继续审查的条目。
5. 保存 TXT。
6. 回到主界面点击 `Start Phase Two`。
7. 在勾选界面里再次确认、选择备份/回收站选项后执行。

## 当前扫描范围

- 第三方启动项
- 浏览器缓存、微信缓存、QQ 临时目录、系统临时目录
- HuggingFace、PyTorch、pip、Conda、npm 等开发缓存
- 下载目录半年未活跃的安装包、压缩包、较大文件
- 桌面半年未活跃的安装包、压缩包、较大文件
- 文档目录半年未活跃的安装包、压缩包

## 当前安全机制

- Windows 目录硬拦截
- 驱动相关目录硬拦截
- 系统盘根目录硬拦截
- 执行前再次检查路径是否仍然安全
- 可选备份
- 可选回收站模式
- 自动生成删除报告

## 不适合直接交给这个工具处理的内容

- 微软系统核心目录
- 驱动目录
- 你不了解用途的软件安装目录
- 普通文档、照片、工作资料这类个人文件
- 需要复杂依赖分析的大型开发环境

## 免责声明

本工具会请求管理员权限，并可能删除无法恢复的文件。请只在你理解风险的前提下使用，并在第二阶段执行前认真检查最终勾选列表。
